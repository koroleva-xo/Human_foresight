# HF Diagnostic Bot

Телеграм-бот для самостоятельного прохождения четырёх диагностических тестов Human Foresight:
Такман (базовый и расширенный), Хофстеде (базовый и расширенный).

Self-study инструмент. Без сбора и сохранения результатов, без продажной механики.
Каждое прохождение разовое: если бот перезапустится, незавершённый тест сбрасывается.

## Файлы

- `data.py` — тексты сценариев, варианты ответов, ключи подсчёта, интерпретации
- `bot.py` — логика бота (aiogram 3)
- `requirements.txt` — зависимости
- `.env` — токен бота (не публикуется, в `.gitignore`)
- `.env.example` — шаблон без токена

## Запуск локально

```bash
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python3 bot.py
```

Токен уже в `.env`, python-dotenv подхватит его автоматически. Бот работает через polling,
держать процесс запущенным пока нужен бот.

## Деплой на постоянный хостинг

Для работы 24/7 нужен постоянный хостинг, локальный компьютер для этого не годится.

### Railway (проще всего)
1. Создать репозиторий на GitHub, залить туда всё кроме `.env` (он в `.gitignore`, это нормально)
2. На railway.app: New Project → Deploy from GitHub repo
3. В Variables добавить `BOT_TOKEN` со значением из `.env`
4. Start command: `python3 bot.py` (Railway обычно подхватывает автоматически по `requirements.txt`)

### Render
1. Тот же репозиторий на GitHub
2. New → Background Worker (не Web Service, боту не нужен HTTP-порт)
3. Build command: `pip install -r requirements.txt`
4. Start command: `python3 bot.py`
5. Environment → добавить `BOT_TOKEN`

### VPS (свой сервер)
```bash
git clone <репозиторий>
cd hf_bot
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
echo "BOT_TOKEN=<токен>" > .env
nohup python3 bot.py &
```
Для автозапуска после перезагрузки сервера лучше systemd-юнит вместо `nohup`, могу собрать
по запросу.

## Важно про токен

Токен в `.env` рабочий. Если репозиторий на GitHub будет публичным, `.env` туда не попадёт
благодаря `.gitignore`, но проверь перед пушем, что файл действительно не закоммичен
(`git status` не должен показывать `.env`). Если токен всё же куда-то утёк, перевыпусти его
через @BotFather командой `/revoke`.

## Известное допущение в данных

Баллы для 15 новых сценариев Хофстеде Extended (по 5 на каждое измерение сверх трёх, общих
с базовым тестом) реконструированы по смыслу ответа, а не взяты из готовой таблицы:
в исходном docx таблица с баллами для этих сценариев отсутствовала. Реконструкция
подтверждена автором методологии. Если позже найдётся оригинальный ключ, скорректировать
словари `HOFSTEDE_EXT_EXTRA` в `data.py`.
